# gatekeep 0.1.0 workspace audit

This is the tracked pre-release artifact for closing the spec-to-workspace gap
before publishing `gatekeep` 0.1.0.

## Closed in the current workspace

- Core decision model: typed facts, presence-only fact sets, lattice outcomes,
  source-order traces, obligations, denial shapes, and stable reason codes.
- Partial evaluation: principal-known facts reduce to residual policies, and
  completion is tested against direct evaluation.
- SQLx adapter: residual lowering, Postgres bind typing, grade projection, and
  Docker-backed differential tests wired through `make test-db`.
- Adapter seams: typed resolver/lowering/audit errors, no-op defaults, identity
  reason catalog, and test-only in-memory audit capture.
- Boundary validation: owned identifiers, locales, facts, and subject references
  reject empty or invalid wire values.
- send-app acceptance model: core tests cover graded case reads, participant
  facts, hidden denial shape, break-glass obligations, and authorized-list
  partial evaluation.

## Remaining 0.1.0 gaps

- `gatekeep-fluent`: implement `ReasonCatalog` over Fluent bundles, including
  missing-message fallback to stable reason codes.
- `gatekeep-keepsake`: map keepsake relations into gatekeep facts without making
  core depend on keepsake.
- `gatekeep-axum`: likely useful for the immediate axum consumer, but keep it
  thin: extractor/middleware glue around caller-owned resolver, policy selection,
  audit sink, and reason catalog.
- Examples: add a runnable axum authorized-list example that exercises SQLx
  partial evaluation and localized denials.
- Workspace shape: add the missing crates only when each has its own tests and
  a real integration boundary; do not publish placeholder crates.

## Release gate

Run these before tagging 0.1.0:

- `make check`
- `make test-db`
- `cargo publish -p gatekeep --dry-run`
- `cargo publish -p gatekeep-sqlx --dry-run`
